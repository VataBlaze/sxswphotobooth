# SXSW Vaporwave ₿ Photobooth

A DIY photobooth built on [photobooth-app](https://github.com/photobooth-app/photobooth-app) (v8.7.0) with a vaporwave + Bitcoin visual identity and a custom **Breathing Session** feature (before-photo → guided breathwork → after-photo).

## Hardware

- **Raspberry Pi 5** (64-bit Raspberry Pi OS — Bookworm or Trixie)
- Touchscreen monitor (min 1024×600)
- Camera — Pi Camera Module, DSLR (gphoto2), or USB webcam
- Dye-sublimation printer (DNP DS-RX1HS, Canon Selphy, Mitsubishi CP, etc.)

## Quick Start

```bash
# 1. Clone this repo
git clone https://github.com/VataBlaze/sxswphotobooth.git
cd sxswphotobooth

# 2. Run the setup script (installs deps, fonts, photobooth-app from PyPI)
chmod +x setup.sh
./setup.sh

# 3. Start the photobooth for first-time configuration
cd ~/photobooth-data
photobooth
# Open http://localhost:8000 → Admin Center → configure your camera

# 4. Run hardware diagnostics to verify camera + printer
bash /path/to/sxswphotobooth/scripts/diagnose-hardware.sh

# 5. Set up your printer (see printer-setup.md)

# 6. Deploy as a boot service with kiosk mode
cd /path/to/sxswphotobooth
chmod +x deploy/*.sh scripts/*.sh
bash deploy/install-service.sh

# 7. Reboot — the photobooth launches automatically
sudo reboot
```

## Architecture

This is a **standalone customization repo** — it contains only the theme files, plugin, scripts, and configuration that sit on top of the upstream photobooth-app. The photobooth-app itself is installed from PyPI as a dependency. There is no fork relationship needed with the upstream repo.

## What's Included

| File / Directory | Purpose |
|---|---|
| `setup.sh` | One-shot system setup for a fresh Pi OS install |
| `userdata/private.css` | Vaporwave ₿ theme (neon gradients, pixel fonts, scanlines) |
| `userdata/breathing.html` | Self-contained Breathing Session page (with logging + API auto-discovery) |
| `userdata/breathe-button.js` | Injects the BREATHE ₿ button on the photobooth frontpage |
| `userdata/frames/vaporwave-btc-frame.png` | Neon grid frame overlay (generated by script) |
| `plugins/breathing_session/` | Server-side plugin with file logging and optional REST endpoints |
| `scripts/patch-breathe-button.sh` | Patches the installed index.html to load the BREATHE button |
| `scripts/diagnose-hardware.sh` | Detects cameras and printers, prints config instructions |
| `scripts/generate-frame.py` | Generates the frame overlay PNG via Pillow |
| `deploy/` | systemd service, kiosk autostart, install script |
| `printer-setup.md` | Printer configuration guide (CUPS + lp commands) |

## Theme

The visual identity fuses **vaporwave** aesthetics (neon pink/cyan/purple, wireframe grids, palm silhouettes, CRT scanlines) with **Bitcoin** motifs (₿ symbol, orange #F7931A, Lightning bolt, "STACK SATS").

Custom fonts (all SIL Open Font License, self-hosted):
- **Press Start 2P** — headings and buttons
- **VT323** — body and status text
- **Monoton** — countdown display
- **Space Mono** — code / secondary text

## Breathing Session

A before/after breathwork experience:

1. Tap **BREATHE ₿** on the main screen
2. A "before" photo is captured automatically
3. A 4-minute guided breathing session plays (4s inhale → 4s hold → 6s exhale → 2s pause)
4. An "after" photo is captured at the end
5. Both photos are displayed side-by-side for review and printing

The session page lives at `http://localhost:8000/userdata/breathing.html` and accepts URL parameters:
- `?duration=300` — session length in seconds (default 240)
- `?pattern=4-4-6-2` — inhale-hold-exhale-pause in seconds
- `?title=BREATHE%20₿` — custom title text

## Logging

All breathing session activity is logged to `~/photobooth-data/log/breathing_session.log`:

- **Plugin-side**: Python rotating file handler (2 MB max, 3 backups). Logs session lifecycle, capture events, errors, and configuration on startup.
- **Frontend-side**: The breathing.html page POSTs log entries to the plugin endpoint and also displays an on-screen log panel (tap the "LOG" button in the bottom-left corner).
- **View logs**: `tail -f ~/photobooth-data/log/breathing_session.log`
- **View app logs**: `journalctl --user -u photobooth-app -f`

## Hardware Diagnostics

If the camera or printer isn't working, run:

```bash
bash scripts/diagnose-hardware.sh
```

This checks for Pi Camera Modules (libcamera), DSLRs (gphoto2), USB webcams (V4L2), and CUPS printers, then prints the exact Admin Center settings you need to apply.

## Configuration

All photobooth-app settings are managed via the **Admin Center** at `http://localhost:8000`. Key areas:

- **Camera**: Admin Center → Configuration → Camera
- **Printer**: Admin Center → Configuration → Share (see [printer-setup.md](printer-setup.md))
- **Theme**: Edit `~/photobooth-data/userdata/private.css`
- **Frame overlay**: Place PNGs in `~/photobooth-data/userdata/frames/`
